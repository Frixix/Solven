# Requisitos del Proyecto

## Requisitos funcionales iniciales

1. El usuario debe poder registrar una transacción.
2. El usuario debe poder elegir si la transacción es ingreso o gasto.
3. El usuario debe poder ingresar monto, categoría, fecha y descripción.
4. El usuario debe poder visualizar todas las transacciones registradas.
5. El usuario debe poder eliminar transacciones.
6. El sistema debe calcular automáticamente el balance total.
7. El sistema debe guardar la información localmente.
8. El sistema debe mantener los datos al recargar la página.

## Requisitos no funcionales iniciales

1. La interfaz debe ser simple y clara.
2. La app debe ser responsive.
3. El código debe estar organizado en archivos separados.
4. El proyecto debe poder escalar a React sin reescribirse de forma caótica.
5. La estructura de datos debe facilitar una futura integración con Supabase.

## Requisitos futuros

1. El usuario podrá editar transacciones.
2. El usuario podrá filtrar por fecha, categoría y tipo.
3. El usuario podrá iniciar sesión.
4. El usuario podrá sincronizar información entre dispositivos.
5. El usuario podrá visualizar resúmenes mensuales y gráficas.
